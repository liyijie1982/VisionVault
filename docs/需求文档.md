# SkyBase 需求文档

## 1. 文档说明

### 1.1 文档目的

本文档用于定义 `SkyBase` 的完整产品需求、业务边界、系统职责、数据模型、接口能力与技术实现约束，作为后续使用 `Golang` 开发 SkyBase 的统一依据。

本文档输出目标：

- 明确 SkyBase 在整个文件采集系统中的定位
- 明确与 `SkyView`、`SkyDrop`、`SkyDropAutoUpdate` 的协作边界
- 明确后台通用管理能力与采集调度能力
- 明确核心业务流程、数据模型和接口分组
- 明确 Golang 版本的架构建议、优先级与实施范围

### 1.2 文档范围

本文档仅覆盖 `SkyBase` 服务端，不覆盖：

- SkyView 前端 UI 视觉设计细节
- SkyDrop Agent 本地执行细节实现
- SkyDropAutoUpdate 本地守护进程实现细节

本文档会定义它们与 SkyBase 的交互要求。

### 1.3 术语说明

- `SkyBase`：文件采集系统的服务端管理与调度中心
- `SkyView`：文件采集系统前端管理页面
- `SkyDrop`：部署在终端主机上的采集 Agent
- `SkyDropAutoUpdate`：SkyDrop 的守护升级程序
- `Agent`：泛指 SkyDrop 实例
- `Storage`：存储目标，支持本地存储与 S3 兼容对象存储
- `Group`：Agent 分组，也是策略聚合中心
- `Policy`：下发给 Agent 的执行策略集合

## 2. 产品定位

### 2.1 总体定位

SkyBase 是文件采集系统的管理调度服务器端，既是一个通用企业应用管理后台，也是一套面向文件采集场景的控制平面。

SkyBase 需要同时承担两类职责：

1. 通用后台职责
   - 用户管理
   - 部门管理
   - 角色管理
   - 菜单管理
   - 系统设置

2. 文件采集平台职责
   - Agent 注册、心跳、监控与调度
   - 采集策略与扫描策略管理
   - 存储配置与文件浏览检索
   - 同步日志、扫描结果、任务进度与告警管理
   - Agent 版本升级与 License 管理

### 2.2 系统角色关系

```text
SkyView  <----HTTP/JSON---->  SkyBase  <----HTTP/JSON---->  SkyDrop
                                             ^
                                             |
                                      SkyDropAutoUpdate
```

关系说明：

- SkyView 通过后台接口管理配置、查看日志、浏览文件、查看监控和告警
- SkyDrop 定期向 SkyBase 拉取策略并提交同步/扫描结果
- SkyDropAutoUpdate 向 SkyBase 查询最新版本并下载升级包
- SkyBase 统一访问 MySQL、Redis、对象存储、Elasticsearch、消息通道

### 2.3 核心价值

- 统一管理全量采集终端
- 统一管理多种目标存储
- 统一定义采集、扫描、过滤、标签、生命周期与工作时间策略
- 统一提供文件浏览、检索、下载与审计能力
- 统一提供监控、告警、授权与升级能力

## 3. 建设目标

### 3.1 业务目标

- 支持企业内多台终端的文件采集与集中管理
- 支持按照部门、业务线、区域或网络范围进行 Agent 分组管理
- 支持从本地磁盘采集文件并同步到指定目标存储
- 支持目录扫描统计，用于摸底、盘点与报表分析
- 支持在后台检索、浏览、下载与审计采集后的文件
- 支持对 Agent 运行状态、资源状态、失联状态进行告警
- 支持 Agent 的自动升级和版本管控

### 3.2 技术目标

- 使用 `Golang` 完成 SkyBase 开发
- 提供清晰的领域边界，避免单体超大 Service
- 满足多 Agent 并发心跳、任务提交和文件检索场景
- 支持后续横向扩展与模块拆分

## 4. 相关系统边界

### 4.1 SkyBase

职责：

- 提供统一后台管理能力
- 提供 Agent 调度与策略下发能力
- 提供存储、文件、日志、告警、升级、授权能力

不负责：

- 前端页面渲染
- Agent 本地采集执行
- 守护进程本地重启与替换文件

### 4.2 SkyView

职责：

- 提供后台管理 UI
- 展示看板、列表、表单、日志、监控和报表

依赖 SkyBase：

- 所有业务数据和操作能力均由 SkyBase 提供 API

### 4.3 SkyDrop

职责：

- 定期发送心跳
- 根据 SkyBase 下发的策略执行扫描与同步
- 向 SkyBase 提交同步日志与扫描结果

依赖 SkyBase：

- 获取存储配置、路径配置、过滤规则、标签规则、工作时间、运行周期、状态控制

### 4.4 SkyDropAutoUpdate

职责：

- 轮询 SkyBase 查询新版本
- 下载升级包
- 停止旧服务、替换 Agent 可执行文件、重新启动

依赖 SkyBase：

- 提供版本查询接口
- 提供升级包下载接口

## 7. 功能需求

## 7.1 基础平台模块

### 7.1.1 用户管理

功能要求：

- 支持用户新增、编辑、删除、禁用、重置密码
- 支持按用户名、手机号、状态查询
- 支持用户绑定部门与角色
- 支持用户登录与会话鉴权
- 支持最后登录时间记录

关键字段：

- 用户名
- 昵称
- 手机号
- 邮箱
- 密码摘要
- 状态
- 部门 ID
- 角色集合

### 7.1.2 部门管理

功能要求：

- 支持树形组织结构
- 支持部门新增、编辑、删除
- 支持负责人设置
- 支持部门状态管理

### 7.1.3 角色管理

功能要求：

- 支持角色新增、编辑、删除
- 支持角色菜单授权
- 支持角色按钮权限授权
- 支持数据权限范围定义

### 7.1.4 菜单管理

功能要求：

- 支持目录、菜单、按钮三种类型
- 支持前端路由字段管理
- 支持排序、显隐、启停

### 7.1.5 系统设置

功能要求：

- 支持系统名称、Logo、版权信息设置
- 支持默认密码策略设置
- 支持 SMTP、企业微信等消息通道全局参数设置
- 支持 Elasticsearch、Redis、对象存储等外部配置管理
- 支持运行参数与功能开关管理

---

## 7.2 存储管理

### 7.2.1 功能目标

统一管理 SkyBase 可访问的文件目标存储，用于：

- Agent 文件同步
- 后台文件浏览与下载
- 日志文件读取
- 批量下载与落盘

### 7.2.2 支持类型

- 本地存储 `local`
- S3 兼容对象存储 `s3`

### 7.2.3 管理能力

- 存储新增、编辑、删除、查询
- 名称唯一性校验
- 连通性校验
- 本地目录树浏览
- 存储配额设置
- 存储状态启停

### 7.2.4 关键字段

- `id`
- `name`
- `type`
- `endpoint`
- `ak`
- `sk`
- `bucket`
- `region`
- `localPath`
- `quota`
- `desc`
- `status`

### 7.2.5 业务规则

- 存储名称必须唯一
- 已被 Agent 分组引用的存储不可直接删除
- 敏感字段如 `sk` 必须加密保存
- 本地存储访问路径必须受系统白名单约束

## 7.3 Agent 管理

### 7.3.1 功能目标

统一管理所有 SkyDrop 节点，展示在线状态、版本状态、资源状态和策略归属。

### 7.3.2 管理能力

- Agent 列表查询
- Agent 详情查看
- Agent 手工新增
- Agent 自动注册
- Agent 编辑
- Agent 删除
- Agent 启停控制
- Agent 迁组
- Agent 标签管理
- Agent 版本查看
- Agent 最近心跳时间、最近提交时间展示

### 7.3.3 关键字段

- `hostSn`
- `ip`
- `status`
- `lastAccessTime`
- `lastCommitTime`
- `groupId`
- `storageId`
- `pathPrefix`
- `version`
- `tags`
- `hostName`
- `remark`

### 7.3.4 业务规则

- Agent 首次心跳时若不存在则允许自动注册
- 自动注册前必须校验 License 可用数量
- 可按 `ipRange` 将 Agent 自动归组
- 已删除 Agent 的历史日志、监控和统计数据应保留

---

## 7.4 Agent 分组与策略中心

### 7.4.1 功能目标

Agent 分组是 SkyBase 的策略聚合根，负责统一下发 Agent 运行时策略。

### 7.4.2 分组能力

- 分组新增、编辑、删除、查询
- 分组启停
- Agent 批量绑定分组
- 根据 IP 范围自动匹配分组

### 7.4.3 分组配置项

- 基础配置
  - 名称
  - 描述
  - 状态
  - IP 范围

- 存储策略
  - 存储 ID
  - 目标路径前缀

- 采集策略
  - 目录列表
  - 目录类型
    - `sync`
    - `scan`
  - 执行周期 `runTime`
  - 最大并发数 `maxWorkers`

- 过滤策略
  - 过滤规则 ID

- 标签策略
  - 正则规则 ID
  - 是否作为路径

- 生命周期策略
  - 删除保留天数 `delTime`

- 工作时间策略
  - `workStartTime`
  - `workEndTime`

- 告警策略
  - 告警组 ID

- 图片处理策略
  - 图片处理规则 ID

- 日志策略
  - 是否开启同步日志索引

### 7.4.4 业务规则

- 一个 Agent 同时只能归属一个分组
- 一个分组必须绑定一个主存储
- 目录列表需要结构化保存，不能仅以字符串拼接
- 分组策略变更后，Agent 下次心跳应立即感知

### 7.7.2 管理能力

## 7.8 文件浏览与检索

### 7.8.1 功能目标

SkyBase 需要作为统一文件访问入口，支持对本地存储与对象存储中的文件进行浏览、检索、预览、下载、删除与批量处理。

### 7.8.2 浏览能力

- 存储列表浏览
- 目录树浏览
- 文件分页列表
- 面包屑路径展示
- 排序
- 条件筛选

### 7.8.3 检索能力

- 文件名搜索
- 路径搜索
- 元数据搜索
- 标签搜索
- 全文/日志关联搜索

### 7.8.4 文件操作能力

- 文件详情查看
- 文件预览
- 单文件下载
- 批量打包下载
- 下载到服务器指定目录
- 上传文件
- 删除文件
- 生成临时访问链接

### 7.8.5 权限要求

- 用户仅能看到授权存储
- 删除、下载、上传等动作要单独校验
- 所有文件操作要记录审计日志

### 7.8.6 业务规则

- 本地与 S3 兼容存储通过统一 Provider 抽象
- 大文件下载应采用流式输出
- 批量下载应走异步任务或临时缓存请求参数
- 删除操作需有操作审计与失败回滚策略

## 7.10 同步日志与文件审计

### 7.10.1 Agent 同步日志

用于记录 Agent 的每次同步任务执行结果。

至少包含：

- Agent 标识
- 路径
- 任务开始时间
- 文件数
- 文件大小
- 错误数
- 日志文件路径
- 提交时间

### 7.10.2 文件审计日志

用于记录后台用户对文件的操作行为。

至少包含：

- 用户
- 存储
- 文件路径
- 操作类型
  - 下载
  - 删除
  - 上传
  - 批量下载
  - 下载到服务器
- 操作时间
- 操作结果
- 客户端 IP

### 7.10.3 日志索引

当分组开启日志索引时，SkyBase 需要：

- 读取 Agent 生成的日志文件
- 逐条解析文件记录
- 写入 Elasticsearch 文件索引

索引至少包含：

- Agent IP
- Agent 标签
- 源路径
- 目标存储名称
- 目标路径
- 文件名
- 文件类型
- 文件大小
- 文件标签
- 文件修改时间
- 上传时间
- 上传状态
- 错误信息

## 7.12 告警系统

### 7.12.1 功能目标

根据 Agent 监控数据与在线状态自动触发告警，并按分组和消息通道发送通知。

### 7.12.2 核心对象

- 告警组
- 告警规则
- 消息通道
- 告警日志

### 7.12.3 支持规则

- CPU 使用率阈值
- 内存使用率阈值
- 磁盘使用率阈值
- CPU 连续超标次数
- 内存连续超标次数
- 心跳超时阈值
- 发送频率限制

### 7.12.4 支持通道

- 邮件
- 企业微信

### 7.12.5 处理流程

1. 定时任务遍历 Agent
2. 读取最新监控数据
3. 根据 Agent 所属分组找到告警组与规则
4. 判断是否满足告警条件
5. 进行频控与抑制
6. 发送告警
7. 写入告警日志

### 7.12.6 业务要求

- 频控状态需持久化，不能仅保存在内存
- 告警发送失败需记录原因
- 支持按 Agent、分组、规则查看告警历史

### 7.14.2 能力要求

- 读取授权文件

## 8. 核心业务流程

## 8.1 Agent 心跳流程

入口：`POST /agent/heartbeat`

流程：

1. 接收 Agent 上报的 `hostSn`、`ip`、`version`、`cpu`、`mem`、`storage`
2. 校验 License 是否过期
3. 根据 IP 或 HostSn 查找 Agent
4. 不存在则执行自动注册
5. 更新 Agent 在线状态、版本与最后访问时间
6. 保存或更新实时监控数据
7. 根据 Agent 所属分组组装策略
8. 返回 Agent 任务配置

返回配置至少包含：

- `paths`
- `storage`
- `delTime`
- `runTime`
- `workStartTime`
- `workEndTime`
- `pathPrefix`
- `status`
- `tags`
- `filters`
- `maxWorkers`
- `fetchField`
- `tagsKeyList`
- `fetchRegex`
- `tagsAsPath`

## 8.2 Agent 同步提交流程

入口：`GET /agent/commit`

流程：

1. 根据 IP 找到 Agent
2. 写入同步任务日志
3. 更新 Agent 最后提交时间
4. 若开启日志索引则读取 `tarListPath` 对应日志文件
5. 逐条解析写入 Elasticsearch
6. 若绑定图片处理规则则触发异步图片处理任务

## 8.3 Agent 扫描提交流程

入口：`POST /agent/scan/commit`

流程：

1. 接收目录扫描统计结果
2. 记录扫描任务日志
3. 持久化扫描结果
4. 更新报表统计缓存或汇总数据

## 8.4 文件浏览与下载流程

入口：

- 浏览：`GET /sky/s3File/list`
- 搜索：`GET /sky/s3File/search`
- 下载：`POST /sky/s3File/download/{storageId}`
- 批量下载：`POST /sky/s3File/batchDownload`

流程要求：

1. 校验用户权限
2. 识别存储类型
3. 通过对应 Provider 执行浏览或下载
4. 写入文件操作日志
5. 批量任务进入异步队列并回传进度

## 8.5 版本升级流程

入口：

- 查询：`GET /agent/version`
- 下载：`GET /agent/download`

流程：

1. SkyDropAutoUpdate 上报当前版本
2. SkyBase 判断是否有更高版本
3. 返回版本元数据
4. AutoUpdate 下载升级包并校验 MD5
5. 本地完成停服务、替换文件、重启

## 8.6 告警流程

触发方式：

- 定时任务
- 手工触发

流程：

1. 扫描 Agent 最新监控快照
2. 判断资源阈值与失联条件
3. 执行频控与去重
4. 发送告警通知
5. 写入告警日志

## 16. 风险与约束

### 16.1 业务风险

- 现有 Agent 心跳返回结构已固定，SkyBase 必须兼容当前字段
- Agent 提交日志与扫描结构已存在，SkyBase 需要按现有协议兼容
- AutoUpdate 已固定依赖 `/agent/version` 与 `/agent/download` 行为

### 16.2 技术风险

- 文件浏览和对象存储逻辑容易膨胀为超大服务
- ES 索引若同步写入会阻塞主链路
- 告警频控若只放内存，重启后状态会丢失
- 本地存储访问存在越权与路径安全风险

### 16.3 控制建议

- 优先保证兼容已有 Agent 协议
- 将耗时处理异步化
- 将频控、任务进度等状态落 Redis
- 对本地文件操作做严格路径校验

## 17. 验收标准

SkyBase 完成后，至少满足以下验收条件：

- 管理员可完成用户、部门、角色、菜单、系统配置管理
- 管理员可完成存储、Agent、分组、过滤、标签、版本管理
- Agent 可正常自动注册、心跳、拉取策略、提交同步日志、提交扫描结果
- SkyDropAutoUpdate 可正常查询版本并下载升级包
- 用户可浏览授权存储中的文件并执行下载
- 系统可记录关键审计日志
- 系统可展示 Agent 在线状态和基础监控信息
- 系统可按规则触发并发送告警

## 18. 结论

SkyBase 不是单一后台，而是一个同时具备“企业后台基础能力”和“文件采集调度控制平面能力”的核心服务。Golang 版本开发时，应以 `Agent 调度`、`存储抽象`、`文件服务`、`监控告警`、`权限体系` 五条主线组织架构，优先保证与现有 SkyDrop、SkyDropAutoUpdate 协议兼容，再逐步增强检索、报表与扩展处理能力。
